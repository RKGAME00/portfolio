RTOS Trampoline

Caractéristiques du Projet

  • RTOS Open Source : Système temps réel conforme OSEK/VDX
  • Multiplateforme : Support ARM Cortex-M, AVR, x86
  • Déterminisme : Ordonnancement préemptif avec priorités fixes
  • Certification : Adapté aux applications critiques automobile

Développements Réalisés

  • Applications temps réel : Tâches périodiques et apériodiques
  • Gestion des ressources : Mutex, sémaphores, files de messages
  • Drivers matériels : GPIO, timers, UART, SPI, I2C
  • Analyse temporelle : Respect des échéances et latences

Architecture RTOS

Plateformes Matérielles

ESP32 Series

Dual-core Xtensa, Wi-Fi/Bluetooth intégré, FreeRTOS natif, 240MHz

STM32F4 Discovery

ARM Cortex-M4F, 168MHz, FPU, rich peripherals, debug intégré

Arduino-based MCU

ATmega328P, 16MHz, écosystème simplifié pour prototypage rapide

FPGA Development

Logique reconfigurable, traitement parallèle, interfaces haute vitesse

Résultats et Apprentissages

Compétences Acquises

  • Maîtrise des concepts RTOS et ordonnancement temps réel
  • Programmation bas niveau et optimisation de code
  • Interface hardware complexe (protocoles série, bus industriels)
  • Debugging avancé avec outils matériels spécialisés

Métriques de Performance

  • Latence de réponse < 1ms pour applications critiques
  • Consommation optimisée : 10µA en veille profonde (ESP32)
  • Uptime > 99.9% pour systèmes de monitoring continu
  • Débit de données jusqu'à 1MB/s via SPI haute vitesse
